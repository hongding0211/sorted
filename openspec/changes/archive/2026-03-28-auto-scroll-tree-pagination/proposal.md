## Why

当前树形浏览器在条目数量超过面板高度时，会继续维护选中索引，但不会自动把视口滚动到选中项附近。结果是用户在浏览较长目录树、展开多层目录或切换到更靠下的位置时，选中行可能已经离开可见区域，必须靠猜测继续操作。

## What Changes

- 为主界面的 source tree 和设置页的 destination tree 增加和选中项同步的滚动偏移状态。
- 在每次重绘时根据当前选中行和面板可视高度自动调整列表 offset，确保选中项保持在可见区域内。
- 补充针对自动滚动偏移计算的测试，并同步更新 README 中对应的 todo。

## Capabilities

### Modified Capabilities
- `tui-experience-polish`: 长列表树形浏览器在移动到不可见项时会自动滚动，使当前选中项持续可见。

## Impact

- 受影响代码集中在 `src/ui/app.rs` 的树列表渲染状态和列表滚动逻辑。
- 不改变现有按键模型、树展开行为或数据加载逻辑，只增强长列表时的可见性。
