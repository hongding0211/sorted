## Why

当前树形文件浏览在返回上一级时只能先把当前目录手动折叠，再额外移动光标回到父目录，连续浏览多层目录时操作偏慢，也容易让用户在长树里丢失上下文。为主界面和设置页补上“快速回退到上一层并折叠当前层”的一致交互，可以减少按键次数，让目录浏览更顺手。

## What Changes

- 为主界面的 source tree 增加“从已展开目录快速回到父目录，并同时折叠当前目录”的快捷回退行为。
- 为设置页的 destination tree 提供相同的快捷回退行为，保持与主界面一致的目录树操作模型。
- 更新树形浏览帮助文案和测试，明确普通折叠与“回退上一层”在焦点、选中项和展开状态上的结果。

### Modified Capabilities
- `tui-experience-polish`: 树形浏览器需要支持在当前目录层级中快速回退到父目录，并在回退时自动折叠刚离开的目录层。
- `archive-settings-ui`: 设置页的目录树需要沿用相同的快速回退交互，确保目标路径浏览和主界面的 source tree 保持一致。

## Impact

- 受影响代码主要在 `src/ui/app.rs` 的树形按键处理、选中索引同步、帮助提示和相关测试。
- 不引入新的外部依赖，也不改变保存设置、导入确认或目录加载的数据来源。
- 需要补充主界面与设置页树形浏览的交互测试，防止已有展开/折叠行为回退。
