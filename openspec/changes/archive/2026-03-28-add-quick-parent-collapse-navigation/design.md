## Context

Sorted 当前的 source tree 和 settings tree 都采用相同的扁平化树渲染方式，但 `Left` 键只处理“当前选中节点已展开时将其折叠”，没有覆盖“当前节点已经是叶子节点或未展开节点时，快速回到父目录”的路径。因此用户从深层目录回退时，往往要先手动把光标移动回父节点，再执行折叠，连续操作层级较深的目录时按键成本偏高。

这次变更不涉及目录加载、配置持久化或列表渲染架构的重写，核心是补上一条更完整的树导航规则，并让主界面与设置页继续共享同一套交互心智模型。

## Goals / Non-Goals

**Goals:**
- 让 `Left` 在树形浏览里同时支持“折叠当前已展开节点”和“快速回退到父节点并折叠父层”两种常见回退路径。
- 保持 source tree 与 settings tree 的导航规则一致，避免同一产品里出现两套目录回退语义。
- 明确回退后选中项、展开状态和帮助文案的预期结果，便于后续测试覆盖。

**Non-Goals:**
- 不新增独立快捷键，也不改变 `Right` 展开、上下移动、`Enter` 确认等现有行为。
- 不重构目录树数据结构或异步加载机制。
- 不改变设置保存、目标路径校验和导入确认流程。

## Decisions

### Extend `Left` into a two-stage back-navigation rule
`Left` 继续保留当前行为的第一优先级: 如果当前选中目录本身处于展开状态，就只折叠该目录并保持选中在该目录上。  
如果当前选中项不是已展开目录，但它存在可见父目录，则执行第二阶段行为: 将选中切换到最近父目录，并同时把这个父目录从展开集合中移除。这样回退完成后，刚刚离开的这一层会被整体折叠，用户能一步回到更高层级。

备选方案是仅把选中移到父节点而不折叠父节点。这个方案虽然实现简单，但仍然保留了深层子节点占据列表空间的问题，无法达到“回退上一层并收起这一层”的目标，因此不采用。

### Derive the parent target from the flattened visible entries
当前应用状态里已经维护了扁平化后的可见树列表和每个条目的 `depth`/`path` 信息。快速回退时，可以基于当前选中项在可见列表中的位置，向上查找最近一个 `depth` 更小的可见祖先项作为父目录目标，再更新展开集合和选中索引。

备选方案是完全依赖 `Path::parent()` 直接计算父路径。这个方案在文件系统层面可行，但不保证父节点当前一定是树里可见且对应同一浏览根，也会让 source tree 的设备根节点与 settings tree 的浏览根处理分叉，因此不采用。

### Keep separate helpers for source tree and settings tree, but share the interaction semantics
主界面和设置页本来就维护不同的展开集合、条目列表和选中索引，因此实现上仍应分别更新各自状态；但应抽象出一致的“左移回退”判定规则，确保未来不会一个界面支持父层折叠，另一个界面仍停留在旧语义。

备选方案是只先改 source tree，等后续再补 settings tree。这样会立即破坏设置页规范中“使用与主界面相同交互模型”的承诺，因此不采用。

### Update contextual help to describe the back-navigation behavior
帮助文案需要明确告诉用户 `Left` 不再只是“collapse folders”，而是支持“collapse or go to parent”。否则新行为虽然存在，用户却难以发现，也会增加对“选中为什么跳走了”的困惑。

## Risks / Trade-offs

- [父节点查找依赖当前可见列表，如果树在重建后索引变化，可能出现选中错位] -> 在重建后按目标父路径重新定位索引，而不是沿用旧索引。
- [设备根节点或设置根节点没有更高父级，`Left` 行为可能让用户误以为失效] -> 在根节点上保持 no-op，并通过帮助文案把行为描述为“回到父层（如果存在）”。
- [新增回退语义后，现有用户可能依赖旧的“左键只折叠当前节点”直觉] -> 保留“已展开节点优先折叠自己”的第一阶段规则，尽量符合常见树控件习惯。

## Validation

- 为 source tree 增加测试，覆盖“选中展开节点时左键只折叠自己”和“选中子节点时左键回到父节点并折叠父层”。
- 为 settings tree 增加对等测试，确认其与主界面遵循相同规则。
- 运行 `openspec status --change add-quick-parent-collapse-navigation`，确认 design/spec/tasks 均处于完成状态后再进入实现阶段。
