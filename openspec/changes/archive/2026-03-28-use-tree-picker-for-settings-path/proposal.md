## Why

当前设置界面的目标路径依赖就地文本编辑，和主界面已经提供的 source 树状浏览体验不一致，也让用户在选择已有目录或逐层确认路径时更容易出错。把设置页的路径选择改成和外部 source 一样的树状结构，可以统一交互模型，并降低手动输入路径带来的负担。

## What Changes

- 将设置界面的 `Destination Root` 从纯文本编辑改为树状目录浏览与选择流程，交互方式与主界面的 source browser 保持一致。
- 在设置界面中提供目录展开、折叠、进入子目录和确认当前目录作为目标路径的能力，并继续保留清晰的焦点与帮助提示。
- 让设置页的验证与保存逻辑适配新的目录选择流程，确保选中的路径仍然沿用现有的可创建、可写校验规则。
- 明确设置页在浏览目录、尚未确认选择、以及保存成功后的反馈行为，避免用户不清楚当前生效的是浏览中的路径还是已保存的路径。

### New Capabilities

### Modified Capabilities
- `archive-settings-ui`: 设置工作流中的目标路径配置方式改为树状目录浏览与选择，并明确对应的焦点、确认和保存反馈要求。

## Impact

- 受影响代码主要包括 `src/ui/app.rs` 中的设置页绘制、焦点管理、按键处理和目录树状态管理逻辑。
- 现有配置持久化与目标路径校验逻辑需要继续复用，但要适配“浏览中候选路径”和“已保存设置值”的区分。
- 需要补充设置页交互与状态反馈相关测试，确保不会回退主界面已有的 source tree 浏览体验。
